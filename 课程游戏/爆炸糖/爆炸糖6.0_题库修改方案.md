# 爆炸糖6.0 题库修改方案

## 1. 目标

本方案用于整理爆炸糖 6.0 版本中题库系统的升级思路，覆盖导入、配置、生成、展示、调整和兼容等环节。

核心目标如下：

1. 导入 Excel 后，不再直接落库，而是先进入列映射配置。
2. 支持筛选列、题目列、答案列的明确区分。
3. 题库从平面结构升级为分层树结构。
4. 支持筛选列顺序调整，并可重新生成题库树。
5. 保留旧题库数据格式，确保兼容。

---

## 2. 现有问题

旧版题库方案存在几个明显限制：

1. 导入流程单一，用户无法在导入后重新定义列角色。
2. 题库组织方式偏平面，不利于按年级、篇目、类型等维度分层。
3. 配置后无法调整筛选维度顺序，灵活性不足。
4. 新旧题库混在一起展示，缺少清晰的区分。

---

## 3. 总体方案

整体采用“导入配置 - 树形生成 - 分区展示 - 顺序调整 - 兼容保留”的思路。

### 3.1 导入配置

导入 Excel 后先弹出列映射对话框，由用户确认：

- 哪些列是筛选列。
- 哪一列是题目列。
- 哪一列是答案列。

默认规则为：

- 倒数第二列为题目列。
- 最后一列为答案列。
- 其余列默认作为筛选列。

### 3.2 题库生成

根据筛选列数量自动生成 N 级树结构，叶子节点保存题目与答案集合。

### 3.3 分区展示

将生成题库与旧式手工题库分开展示：

- 生成题库：按筛选层级分组显示。
- 传统题库：保留原有展示方式。

### 3.4 顺序调整

支持拖拽调整筛选列优先级，并根据新顺序重新生成树结构。

### 3.5 兼容保留

保留旧数据结构和旧的导入/导出逻辑，确保历史数据可继续使用。

---

## 4. 数据结构设计

### 4.1 列配置状态

```javascript
columnConfigState = {
  jsonData: [],
  headers: [],
  filterCols: [],
  questionCol: -1,
  answerCol: -1
}
```

### 4.2 生成题库配置

```javascript
qbConfig = {
  configName: '按级别和篇目分类',
  filterCols: [0, 1],
  questionCol: 2,
  answerCol: 3,
  levelCount: 2,
  createdAt: '2026-04-16T00:00:00.000Z'
}
```

### 4.3 题库树结构

```javascript
{
  '初中': {
    '古文': {
      _items: [
        { q: '问题1', a: '答案1' },
        { q: '问题2', a: '答案2' }
      ],
      _count: 2
    }
  }
}
```

### 4.4 本地存储约定

- `mc_qb_configs`：保存生成题库配置。
- `mc_current_qb_config`：保存当前选中的生成配置。
- `mc_yly_history`：保留旧题库历史数据。

## 5. 关键实现模块

### 5.1 Excel 导入入口

导入逻辑改为：

1. 点击导入按钮。
2. 选择 Excel 文件。
3. 使用 SheetJS 解析为二维数组。
4. 检查数据行是否有效。
5. 打开列映射对话框。

### 5.2 列映射对话框

对话框采用三块式拖拽界面：

- 左侧：筛选列。
- 右侧上方：题目列。
- 右侧下方：答案列。

支持的交互方式：

- 筛选列内部拖拽排序。
- 题目列与答案列互换。
- 实时更新配置摘要。

### 5.3 题库树生成

生成逻辑按筛选列逐层递归分组：

1. 读取所有数据行。
2. 按第 1 个筛选列分组。
3. 再按第 2 个筛选列分组。
4. 直至最后一层。
5. 在叶子节点存放题目列表。

### 5.4 题库列表展示

生成题库与传统题库分开渲染：

- 生成题库按配置名展示。
- 同一配置下按照层级展开。
- 每个节点显示题目数量。
- 旧题库保留独立区域。

### 5.5 顺序调整

当用户拖动筛选列顺序时：

1. 修改 `filterCols` 顺序。
2. 读取原始 Excel 数据。
3. 重新构建题库树。
4. 刷新侧边栏显示。

## 6. 核心函数清单

### 6.1 showColConfigDialog(jsonData)

作用：初始化列配置状态并显示配置弹窗。

### 6.2 renderColumnConfigDialog()

作用：渲染三块式拖拽界面，重建所有列卡片。

### 6.3 updateConfigSummary()

作用：根据当前配置刷新摘要信息与状态样式。

### 6.4 confirmColConfig()

作用：检查配置有效性，生成题库树并保存到本地。

### 6.5 buildAndSaveQBTree(jsonData, qbConfig)

作用：根据配置生成树形题库并写入 localStorage。

### 6.6 loadQBConfigList()

作用：渲染生成题库配置列表，并按层级数量分组。

### 6.7 loadQBConfig(configName)

作用：加载指定生成配置到题库预览或游戏流程中。

## 7. UI 设计说明

### 7.1 配置弹窗

视觉上分为三种颜色：

- 筛选列：黄色系。
- 题目列：蓝色系。
- 答案列：紫色系。

这样做的目的是让用户一眼区分三个角色，并降低误操作概率。

### 7.2 题库展示区

展示区拆分为两个部分：

1. 生成题库。
2. 传统题库。

这样可以避免新题库和旧手工数据混排。

---

## 8. 验证要点

建议重点检查以下内容：

1. 导入按钮点击后是否能正确打开文件选择器。
2. 选择 Excel 后是否能弹出列配置对话框。
3. 筛选列拖拽是否能实时更新顺序。
4. 题目列和答案列是否支持互换。
5. 配置确认后是否能正确生成题库树。
6. 生成题库与传统题库是否分开显示。
7. 旧题库是否仍能正常使用。

## 10. 总结

本次题库修改的核心，是把原来单一的平面导入方式，升级为可配置、可拖拽、可分层、可扩展的题库系统。方案兼顾了用户体验、数据结构和兼容性，能够支撑后续更多题库玩法的扩展。
