# 华山论剑 - 技能与效果全解析

## 📋 概述

华山论剑游戏共包含 **5个终极技能**，每个技能都配备了独特的**视觉特效（VFX）**、**音效**和**卡片形变动画**。技能在答题正确时随机启动，进行三阶段呈现：
1. **切镜阶段** - 上下斜向黑色蒙层缓缓展开，呈现中间的大招名称毛笔字
2. **视觉冲击** - 屏幕内攻击特效和受击方形变反应
3. **声音反馈** - 多层次音效设计

## 🗡️ 技能详解

### 1️⃣ **一剑开天门**
**代码ID**: 0  
**反应类**: `react-slash`

#### 🎨 视觉特效（VFX）
- **主要元素**：马赛克宝剑发射 + 纵向劈击 + 二分裂纹
- **阶段一（0-0.25s 发射阶段）**：
  - 攻击方屏幕上方：金色马赛克宝剑渐进出现
  - 宝剑采用 8-16 像素马赛克风格绘制
  - 从上往下加速坠落轨迹带金色尾焰
  - 施法方卡片无额外变化
- **阶段二（0.25-0.5s 劈击阶段）**：
  - 宝剑到达受击方，触发竖向劈击动画
  - 受击方卡片从中间产生白色发光裂纹（竖向）
  - 上下两半同步分离动画，扩散距离 20px
  - 若未死亡掉血，两半卡片合并；若死亡则上下两半分别消失
  - 裂纹同步伴随白光闪烁（brightness 2x）

#### 🎵 音效
- **类型**：拔剑高光音效 (二次元 Zing)
- **音频参数**：
  - 主频率：4000Hz → 8000Hz（指数变化，0.1s）
  - 音量：0 → 1 → 0.01（线性上升0.02s，指数下降0.5s）
  - 辅助噪声：高通滤波(4000Hz → 8000Hz，0.4s)
  - 总时长：0.5s

#### 💥 卡片形变动画（受击方）
- **动画名称**：`reactSlash`
- **持续时间**：0.6s
- **阶段变化**：
  - 0-25%：抵抗震颤，scaleX(0.95)，亮度↑ 2x
  - 25-50%：纵向分裂，translateY 分离（上半部+5vh，下半部-5vh），rotate(±2°)
  - 50-100%：合并恢复，translateY 恢复，亮度归一
- **若死亡**：分裂后两半分别置灰（grayscale 100%）并缓降消失
- **特殊效果**：金色阴影投射 `drop-shadow(0 0 30px #F0C239)`

### 2️⃣ **泰山压顶**
**代码ID**: 1  
**反应类**: `react-mountain`

#### 🎨 视觉特效（VFX）
- **主要元素**：我的世界风格马赛克山体坠落 + 停留压制 + 冲击波
- **阶段一（0-0.3s 下落阶段）**：
  - 屏幕上方出现 32x32 像素马赛克山体（棕色层叠）
  - 山体从 top: -50% 快速加速坠落至 top: 20%（ease-in）
  - 山体采用 Minecraft 风格的块状纹理，多层渐进色
- **阶段二（0.3-0.55s 压制阶段）**：
  - 山体停留在受击方上方，保持 0.25s 不动
  - 地面产生 3 层土黄色冲击波圆纹从中心扩散
  - 冲击波边框厚度递减（30px→10px），色值从 `#955539` 渐变
- **阶段三（0.55-0.8s 消退阶段）**：
  - 山体淡出消失（opacity 1→0）
  - 冲击波继续扩散至屏幕边界
  - 若受击方未死亡，卡片保持压扁形态后恢复

#### 🎵 音效
- **阶段1**：下落呼啸（0.3s）
  - 类型：带通滤波噪声 (800Hz → 200Hz)
- **阶段2**（延迟0.3s开始，0.7s）：
  - 低频震鸣：80Hz → 8Hz（线性下降）
  - 音量：2.5 → 0.01（指数衰减）
  - 辅助低通噪声（0.6s）：300Hz → 20Hz
- **总效果**：从高频呼啸→低频重击的听觉冲击

#### 💥 卡片形变动画（受击方）
- **动画名称**：`reactMountain`
- **持续时间**：0.8s（需与山体动画同步）
- **阶段变化**：
  - 0-30%：scaleY(0.2) + scaleX(1.1)，模糊度 12px，亮度 0.3x，保持于山体下方
  - 30-55%：保持压扁状态，scaleY(0.25)，不要与山体重叠（设置卡片z-index低于山体）
  - 55-80%：scaleY(0.6) + scaleX(1.05)，模糊递减，亮度回升，山体消失后卡片快速恢复
  - 80-100%：scaleY(1) scaleX(1)，完全恢复正常
- **若死亡**：80% 时置灰（grayscale 100%），scaleY 保持 0.5 并缓降消失
- **关键约束**：山体下落时卡片不应被山体覆盖，而是在其下方被压扁

### 3️⃣ **碎空锤**
**代码ID**: 2  
**反应类**: `react-shatter-fx`

#### 🎨 视觉特效（VFX）
- **主要元素**：紫色能量锤击 + 卡片旋转飞出
- **锤击特效**：
  - 位置：受击方卡片中心
  - 形状：紫色能量球（`#9C27B0`）+ 放射光线
  - 冲击时离子闪烁效果（高频闪烁 4-5 次）
  - 冲击持续 0.3s，旋转冲击形成的视觉感
- **卡片飞出**：
  - 受击卡片在锤击后旋转飞出
  - 飞出方向：根据受击方位置决定（左侧飞左，右侧飞右）
  - 旋转角度：360°+ 递增（快速旋转）
  - 飞出距离：50-80vw，同时缩放消失
  - 透明度随飞出递减至 0
  - 持续时间：0.6s

#### 🎵 音效
- **类型**：碎裂空间音效
- **参数**：
  - 频率：3000Hz → 200Hz（三角波，指数变化）
  - 音量：1 → 0.01（0.3s衰减）
  - 辅助噪声：带通滤波 (4000Hz → 1000Hz，0.4s)
- **特色**：高频→低频的"碎裂"下滑感

#### 💥 卡片形变动画（受击方）
- **动画名称**：`reactShatter`
- **持续时间**：0.6s
- **阶段变化**：
  - 0-15%：锤击吸收，scaleX(0.8) scaleY(0.9)，亮度 1.5x，紫色发光
  - 15-40%：旋转飞出，translateX ± 60vw，rotate(360° + 180°)，scaleX 1.2
  - 40-100%：继续旋转消失，translateX ± 80vw，rotate(+360°)，scaleY 缩小，opacity 1→0
- **若死亡**：飞出过程中卡片部分置灰，最后完全消失
- **特殊效果**：紫色能量环绕 `drop-shadow(0 0 30px #9C27B0)`，飞出时闪烁

### 4️⃣ **火弩箭**
**代码ID**: 3  
**反应类**: `react-burn`（受击方）/ `react-cast-shake`（施法方）

#### 🎨 视觉特效（VFX）
- **主要元素**：5条火箭连发 + 受击白光穿孔 + 施法方后坐抖动

**施法阶段（0-0.5s 攻击发射）**
  - 位置：攻击方屏幕上方
  - 5 条火箭分次发射（间隔 0.1s）
  - 每条火箭采用橙红渐变 + 尾焰效果
  - 火箭轨迹向下加速，带有抖动的类折线轨迹（模拟不稳定）
  - 配合 5 次连续音效
  - **施法方（左侧或右侧）卡片同步产生后坐力抖动**（见下方形变动画）
  - 发射完成后施法方卡片抖动消失

**受击阶段（0.5-1.0s 抵达与穿孔）**
  - 5 条火箭依次抵达受击方
  - 每次抵达产生突击闪光（white flash，50ms）
  - 在受击方卡片上留下 5 个穿孔：
    - 穿孔位置：呈「5」字排列
    - 穿孔大小：1-2vw 圆形孔洞
    - 穿孔特效：白色发射光线从孔洞射出（放射35条线，0.3s）
    - 孔洞周边烧焦效果（橙红→黑色渐晕）
  - 穿孔依次出现，最后三个穿孔同时闪耀

**若受击方死亡**
  - 穿孔部分会扩大并显示卡片背部
  - 被穿孔处产生崩碎效果并消失

#### 🎵 音效
- **发射音效**（5 连发）：
  - 高通噪声 "咻" ：2000Hz → 6000Hz，每次 0.15s
  - 5 次间隔 0.1s，形成连续箭鸣效果
  - 音量递增（第1-3 次音量较低，第4-5 次音量突增～后坐感）
- **抵达音效**（5 次）：
  - 沉闷击中音：150Hz → 50Hz，每次 0.2s
  - 穿孔时配合小幅爆炸音（mid频段突发）
  - 5 次共计 1.0-1.2s
- **总效果**：快速连发感 + 多次穿孔撞击的密集轰鸣

#### 💥 卡片形变动画

**受击方 (react-burn)**
  - **动画名称**：`reactBurn`
  - **延迟**：0.3s（火箭抵达时）
  - **持续时间**：1.0s
  - **阶段变化**：
    - 0-20%：每次穿孔时产生被击感（scale 0.95→1.0），闪白（brightness 3x）
    - 20-80%：间歇性闪烁（5 次穿孔对应 5 次闪烁）
    - 80-100%：恢复平稳，穿孔效果保留但淡出
  - **若死亡**：80% 时置灰（grayscale 100%），穿孔扩大并产生崩碎效果
  - **效果**：被穿孔的冲击感，卡片保持完整但留下伤痕

**施法方 (react-cast-shake)**
  - **动画名称**：`castShake-fire`
  - **开始时间**：0s（与第一箭发射同步）
  - **持续时间**：0.5s
  - **模式**：限时抖动（非无限循环），共 5-6 个抖动周期
  - **抖动幅度**：
    - 基础抖动：±8px 横向位移，±4px 纵向位移
    - 旋转幅度：±1.5°
    - 每次抖动持续 50-80ms，间隔 80ms
  - **后坐力感**：后 2-3 次抖动强度递减（缓冲感）
  - **消失方式**：抖动完成后卡片恢复平稳，无额外变化
  - **效果**：模拟连发火箭的后坐力，强烈但短暂

### 5️⃣ **雷鸣斩**
**代码ID**: 4  
**反应类**: `react-strobe`

#### 🎨 视觉特效（VFX）
- **主要元素**：三颗天雷球发射 + 受击方不规则裂纹 + 镜面破碎音效

**天雷球发射阶段（0-0.3s）**
  - 位置：攻击方屏幕上方
  - 3 颗天雷球呈倒三角形排列，依次出现（间隔 0.1s）
  - 每颗球采用蓝紫色光球 `#9C27B0` + 闪电纹理
  - 球体大小：2-3vw，内部生成 4-5 条放射闪电纹
  - 下落轨迹：竖直下落，带有轻微左右摆动
  - 放射光晕：drop-shadow 蓝紫色，持续扩张收缩

**破坏阶段（0.3-0.8s）**
  - 3 颗天雷球依次抵达受击方
  - 每次撞击产生白蓝色闪光（200ms）
  - 受击方卡片生成不规则裂纹：
    - 裂纹颜色：淡蓝色 `#ADD8E6` + 白色高光
    - 裂纹形状：从撞击点向外辐射，呈不规则星爆状
    - 裂纹宽度：2-4px，边缘模糊发光
    - 3 次撞击对应 3 组裂纹群，覆盖整个卡片
  - 裂纹伴随频闪白光（亮度 1.5x→1.0x 循环 3 次）
  - 若受击方死亡，裂纹扩大并产生碎片飘散效果

#### 🎵 音效
- **天雷出现音效**（3 次）：
  - 高频电流音：5000Hz → 8000Hz，每次 0.1s
  - 间隔 0.1s，形成三次激活的蓄电感
- **撞击音效**（3 次）：
  - 主雷击音：200Hz → 50Hz，每次 0.4s
  - 配合 **镜面破碎音效**：
    - 玻璃碎裂音（高频噪声叠加，5000Hz 以上）
    - 间歇性尖锐的「咔咔咔」声
    - 共 3-5 次破碎音，0.1-0.2s 间隔
- **余韵**（延迟0.3s）：
  - 低通噪声衰减尾音：1000Hz → 50Hz，0.4s
- **总效果**：三次雷击 + 镜面碎裂的清脆破碎感 + 雷鸣回响

#### 💥 卡片形变动画
- **动画名称**：`reactStrobe`
- **持续时间**：0.8s
- **阶段变化**（三次冲击 + 裂纹效果）：
  - 0-10%：准备阶段，brightness 1.2x，淡蓝色滤镜 (filter: hue-rotate 200°)
  - 10-25%：首次撞击，scaleX(0.9) scaleY(1.05)，brightness 2x，反色化
  - 25-40%：中间恢复，brightness 1.3x，裂纹显现
  - 40-55%：第二次撞击，scaleX(1.1) scaleY(0.95)，brightness 2x+invert 0.5
  - 55-70%：第三次撞击，scaleX(0.95) scaleY(1.1)，brightness 2.5x，白蓝混合
  - 70-100%：平复恢复，裂纹保留但逐渐淡出（opacity 1→0.7）
- **若死亡**：70% 时置灰，裂纹扩大产生碎片效果，卡片碎裂消失
- **裂纹特效**：不规则淡蓝色发光线条，密度逐次增加

## 🎬 通用音效（非技能特定）

### 开局音效
- **类型**：`playSound('start')`
- **特性**：方波音，阶段下滑频率变化

### 准备就绪
- **类型**：`playSound('ready')`  
- **特性**：正弦波上升扫频 (1200Hz→2000Hz)，快速切入淡出

### 击鼓（进度推进音）
- **重鼓**：`playSound('drum_heavy')` - 低频深沉，0.08s
- **轻鼓**：`playSound('drum_light')` - 中频敏捷，0.05s

### 失手（答题错误）
- **类型**：`playSound('miss')`
- **特性**：心脏停跳感的低频音，60Hz开始递减

### 段位升级
- **类型**：`playSound('rankup', rankType)`
  - `rankType='gold'`：8音五声阶梯（C→D→E→G→A→C→E→C），最后钟鸣回响
  - `rankType='silver'`：7音五声阶梯，同样钟鸣
  - `rankType='bronze'`：6音五声阶梯，较短钟鸣
- **特性**：中国五声音阶 + 余韵钟鸣

### 再起复活
- **类型**：`playSound('revive')`
- **特性**：正弦波上扬 (200Hz→800Hz)，混合带通噪声，0.6s

### 获胜膨胀
- **类型**：`playSound('hpburst')`
- **特性**：正弦波上升 (600Hz→1200Hz)，胜利欢快感

  - 名称字体\uff1a朵方 / ZhiMangXing\uff0c尺寸 20-25vh
  - 名称色彩\uff1a金色 `#F0C239` + 朱砂 `#C93756` 双色渐变

### 伤害反馈体系
- **答题正确**：
  - 攻击方卡片：无额外反馈（保持稳定）
  - 受击方卡片：根据技能施加对应形变动画（见各技能详解）
  - 音效层：连续多阶段音效反馈（发动→飞行→着陆→结算）
  
- **答题错误（内伤）**：
  - 攻击方卡片：产生"内伤"效果（黑色向心收缩）
  - 受击方卡片：保持原状，不受击
  - 音效：低频心脏停跳音 + 失败低鸣
  
- **死亡处理**：
  - 受击致死：卡片执行致死形变（具体见技能描述）→ 马赛克/裂纹/消失效果 → 完全置灰 → 缓降消失
  - 复活成功（再起）：播放复活音效 → 卡片从下方淡入恢复 → HP+1
  - 游戏结束：败方卡片保持置灰状态

### 细节音效激活时序（升级版）

## 📊 性能优化关键点

1. **VFX 动画** - 仅在 `gameState === 'ultimate_impact'` 或 `result` 时渲染，确保高效
2. **音频上下文** - Web Audio API 延迟启用，避免浏览器冻结
3. **CSS 动画** - 所有变换使用 `transform` 和 `filter`，避免重排重绘
4. **混合模式** - `screen`/`overlay`/`multiply` 合理应用确保视觉效果同时保持帧率
5. **SVG 路径** - 使用 `data:image/svg+xml` 内联，避免额外网络请求
6. **多层粒子** - 限制粒子数量 (火箭、天雷、裂纹线条合计不超过 20-30 个活跃元素)
7. **z-index 层级** - 明确卡片层级 (VFX 50-100、卡片 10-30、山体 5-15)

### 动画时序协调

- VFX 视觉特效应与音效阶段同步

- 若存活：形变完成后卡片恢复原位，留下可见伤害标记（如穿孔、裂纹）

- **浏览器兼容**：CSS `filter` 和 `backdrop-filter` 保留备选方案
- **移动端**：考虑降低粒子数量和禁用某些高耗能动画
- **音频**：需要用户交互启用 Web Audio Context（首次点击后激活）
