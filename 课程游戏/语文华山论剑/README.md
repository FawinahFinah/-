# 华山论剑 · 武林诗词对战游戏

> 卡通中国风 · 双人实时对战 · 课堂互动答题

---

## 产品简介

**华山论剑**是一款面向语文课堂的双人实时对战答题游戏。两名学生各占半屏，通过抢答诗词填空题来发动武侠大招攻击对方。答对触发随机大招特效，答错自受内伤。胜者留守擂台，败者由老虎机随机抽取新挑战者，循环对战直至计时结束，最终按积分排出武林金榜。

---

## 核心玩法

### 游戏流程

```
开始设置
  ↓ 导入名册（Excel）/ 设置时间
起鼓开战
  ↓
[准备阶段] 双方各点"准备"按钮
  ↓ 双方就绪
[题目揭示] 显示诗词上句（1.5秒）
  ↓
[对战阶段] 答案选项滚动切换（每1.25秒换一次，共8次）
  ↓ 任意一方点击按钮
  ├─ 答对 → [大招切镜] → [大招特效] → 对方扣血
  └─ 答错 → [内伤特效] → 自己扣血
  ↓
[结算]
  ├─ 有人阵亡 → [老虎机] 随机抽取新挑战者，胜者加血
  └─ 无人阵亡 → 直接进入下一轮
  ↓
计时归零 → [武林金榜] 积分排名
```

### 得分规则

| 事件 | 分值 |
|------|------|
| 答对（击败对手）| +10分 |
| 答错（自受内伤）| 0分 |

### 五大绝学

| 技能 | 触发条件 | 受击方效果 |
|------|----------|------------|
| 🗡️ 一剑开天门 | 随机（20%）| 错位闪白 + 剑气粒子 |
| ⛰️ 泰山压顶 | 随机（20%）| 压扁重击 + 冲击波 |

| ☄️ 火 弩 箭 | 随机（20%）| 烧穿底片效果（延迟1.2s）|

## 技术栈

| 类别 | 技术 | 版本/说明 |
|------|------|-----------|
| **UI框架** | React | 18（CDN UMD，无构建工具）|
| **JSX编译** | Babel Standalone | 浏览器内实时编译 |
| **CSS框架** | Tailwind CSS | CDN，utility-first |
| **字体** | Zhi Mang Xing | Google Fonts，毛笔书法风 |
| **字体** | Noto Serif SC | Google Fonts，衬线中文 |
| **音效** | Web Audio API | 原生浏览器 API，程序化合成 |
| **Excel解析** | SheetJS (xlsx) | 0.18.5，动态加载 |
| **运行环境** | 纯浏览器 | 无服务器，双击 HTML 即可运行 |

### 架构特点

- **零依赖构建**：单个 HTML 文件，所有依赖通过 CDN 加载，无需 Node.js / npm
- **程序化音效**：全部音效由 Web Audio API 实时合成（振荡器 + 噪声 + 滤波器），无需音频文件
- **CSS-only 动画**：所有技能特效、背景粒子、UI 动画均为纯 CSS `@keyframes`，无 Canvas/WebGL
- **状态机驱动**：游戏流程由 React state 驱动，状态清晰可追踪

### 游戏状态机

```
setup
  └─→ ready
        └─→ question_reveal
              └─→ battle
                    ├─→ ultimate_cutin → ultimate_impact → result
                    │                                         ├─→ slot_machine → ready（循环）
                    │                                         └─→ ready（循环）
                    └─→ ultimate_impact（答错）→ result → ...
                                                    └─→ gameover（计时归零）
```

## 使用说明

1. 用浏览器（Chrome / Edge 推荐）打开 `华山论剑.html`
2. 可选：点击"名册导入"上传 Excel 文件（需含"姓名"列）
3. 拖动滑块设置决战时间（1~20刻，1刻=1分钟）
4. 点击"起鼓开战"开始游戏
5. 左侧玩家用左半屏按钮操作，右侧玩家用右半屏按钮操作
6. 计时结束后查看武林金榜
