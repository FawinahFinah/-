## 1. 项目概述

### 名称
华山论剑 · 武林诗词 - 古诗词卡通对战游戏

### 核心玩法
- **双人对战**: 两个玩家同屏对阵
- **答题机制**: 基于古诗词互动（上句对下句）
- **终极技能**: 5种华丽的必杀技特效
- **段位系统**: 积分类晋升制度（铜、银、金）
- **复活机制**: 高段位玩家有概率死后复活
- **擂主制度**: 胜者留台挑战新玩家

### 教育目标
- 提升古诗词记忆力
- 增强对文言文的理解
- 通过竞赛形式激发学习兴趣

## 3. 游戏流程状态机

### 状态转换图

```
setup (准备) 
  ├─ 选择题库
  ├─ 导入名单
  └─ 设定时间
      ↓
ready (就绪)
  ├─ 等待两名玩家点击"准备"
  └─ 显示当前题目
      ↓
question_reveal (题目展示)
  └─ 3秒倒计时显示题目
      ↓
battle (战斗)
  ├─ 题目和答案选项轮流弹出
  ├─ 玩家点击进行作答
  └─ 等待8个选项全部出现或其中一人作答
      ↓
ultimate_cutin (大招切镜)
  ├─ 1秒黑色条纹拔剑高光动画
  ├─ 播放"cutin"音效
  └─ 显示技能名称
      ↓
ultimate_impact (大招冲击)
  ├─ 1.2-1.8秒技能VFX动画
  ├─ 播放技能对应音效
  └─ 受击方显示"内伤" 或攻击方显示伤害
      ↓
result (结算)
  ├─ 判定血量变化
  ├─ 检查段位升级
  ├─ 检查是否复活
  └─ 播放段位升级动画（可选）
      ↓
slot_machine (老虎机) [仅胜者选手死亡时]
  ├─ 滚动选择新对手
  └─ 暂停游戏倒计时
      ↓
ready (下一轮就绪)
```

### 5.1 一剑开天门 (ID: 0)

**主题**: 二次元拔剑斩击  
**关键字**: 竖向裂纹、裂纹扩大、二次元高光

#### 视觉效果详解

发射阶段 (0-0.3秒):
- 攻击方展示拨剑操纵，从屏幕上方降下金色宝剑
- 宝剑以拨弦动作逐渐增加光辉
- 宝剑从上方下跌，速度逐步加快

斩击阶段 (0.3-0.8秒):
- 宝剑撇中受击方卡片，出现竖向白色发光裂纹
- 裂纹从卡片中心向下扫过，裂纹不断扩大
- 白色发光伴随裂纹搐动（brightness 2x周期变化）
- 至裂纹扩大到整个卡片，撇撥结束

伤害结果:
- 若未死亡：裂纹消退，卡片恢复正常，仅有淡迹
- 若死亡：裂纹保持、卡片整体置灰上遥消失

CSS类: `.pixel-sword`, `.reactSlash`, `.sword-crack-expand`

#### 音效 (总0.5秒)
```javascript
playSound('cutin')           // 拔剑高光音
playSound('ultimate', 0)     // 剑气音效

// 详细参数：
// 主频率：4000Hz → 8000Hz (指数变化，0.1s)
// 音量：0 → 1 → 0.01 (线性上升0.02s，指数下降0.5s)
// 辅助噪声：高通滤波 4000Hz → 8000Hz (0.4s)
```

#### 卡片形变动画 (受击方)
**动画名称**: `reactSlash`  
**持续时间**: 0.6秒

| 阶段 | 时间 | 变化 | 效果 |
|-----|------|------|------|
| 抵抗震颤 | 0-25% | scaleX(0.95), brightness 2x | 冲击吸收感 |
| 纵向分裂 | 25-50% | translateY ±5vh, rotate(±2°) | 白光裂纹分离 |
| 合并恢复 | 50-100% | translateY归零 | 恢复原样或若已死亡则消失 |

**特殊效果**: 金色阴影 `drop-shadow(0 0 30px #F0C239)`

**伤害计算**:
- 正确作答：对方-1 HP
- 错误作答：自己-1 HP

### 5.2 泰山压顶 (ID: 1)

**主题**: 山石坠落土块埋没  
**关键字**: 山体收复，土黄色土块，重力堆积

#### 视觉效果详解

贯穿阶段 (0-0.3秒):
- 金色宝剑从屏幕上方高速坠落，中心贯穿受击方卡片
- 宝剑流光四溢，伴随剑鸣音效
- 剑身呈剑形通透梯度，金色+白色高光

裂纹阶段 (0.3-0.8秒):
- 宝剑贯穿处沿卡片中间产生竖向白色发光裂纹
- 裂纹逐步扩大，从细线(3px)→宽裂缝(16px)
- 裂纹伴随金色光晕 `drop-shadow(0 0 60px rgba(255,215,0))`
- 裂纹搐动闪烁，给予明显的"被贯穿"感

分裂效果 (0.35-0.95秒):
- 宝剑贯穿后，卡片沿中间裂纹从中心分裂
- 左半部分向左侧飘出，同时向左倾斜(skew -5deg)，逐步消失
- 右半部分向右侧飘出，同时向右倾斜(skew 5deg)，逐步消失
- 伴随金色光晕渐进消退

伤害结果:
- 若未死亡：卡片裂纹消退，恢复完整，仅有淡淡发光迹象
- 若死亡：卡片已分裂，人物形象从中间消失

CSS类: `.minecraft-mountain`, `.reactMountain`, `.dust-blocks-fall`

#### 音效 (总0.7秒)
```javascript
playSound('ultimate', 1)

// 阶段1 (0-0.3s) - 下落呼啸：
// 带通滤波噪声：800Hz → 200Hz

// 阶段2 (0.3-1.0s) - 低频震鸣：
// 主音：80Hz → 8Hz (线性下降，0.7s)
// 音量：2.5 → 0.01 (指数衰减)
// 辅助低通噪声：300Hz → 20Hz (0.6s)
```

#### 卡片形变动画 (受击方)
**动画名称**: `reactMountain`  
**持续时间**: 0.8秒  
**关键约束**: 山体下落时卡片在其下方被压扁，不被山体覆盖

| 阶段 | 时间 | 变化 | 备注 |
|-----|------|------|------|
| 被压 | 0-30% | scaleY(0.2) + scaleX(1.1), blur 12px | 亮度0.3x |
| 保持 | 30-55% | scaleY(0.25) 保持压扁 | z-index低于山体 |
| 恢复 | 55-80% | scaleY(0.6→1), scaleX(1.05→1) | 山体消失后快速恢复 |
| 正常 | 80-100% | 完全恢复 scaleY(1) scaleX(1) | 亮度回升 |

**死亡时**: 80%处置灰，scaleY保持0.5并缓降消失

### 5.3 碎空锤 (ID: 2)

**主题**: 锤子砸中，向外扩散裂纹  
**关键字**: 中心锤印，蓝色裂纹，四周破碎

#### 视觉效果详解

砸下阶段 (0-0.3秒):
- 紫色马赛克风格锤子从屏幕上方高速砸下
- 锤子八边形设计，伴随紫色光晕
- 砸中受击方卡片中心时触发撞击

冲击阶段 (0.3-0.6秒):
- 锤子撞击处产生圆形锤痕，中心向外扩散
- 锤痕从圆心(0px)→中等(150px)→大(200px)，伴随紫色光晕扩大
- 锤痕内侧呈深紫色凹陷感，周围有阴影

碎裂阶段 (0.35-0.7秒):
- 中心锤痕周围产生8条不规则碎裂射线
- 碎裂线条呈紫色发光，向四周不均匀扩散
- 有的碎裂长、有的短，营造破裂感
- 碎裂线条逐步消退、透明消失

伤害结果:
- 若未死亡：碎裂消失后卡片恢复
- 若死亡：卡片整体置灰消失

#### 音效 (总0.6秒)
```javascript
playSound('ultimate', 2)

// 锤击音效：
// 频率：3000Hz → 200Hz (三角波，指数变化)
// 音量：1 → 0.01 (0.3s衰减)
// 辅助噪声：带通滤波 4000Hz → 1000Hz (0.4s)
// 特色：高频→低频的"碎裂"下滑感
```

#### 卡片形变动画 (受击方)
**动画名称**: `react-shatter-fx`  
**持续时间**: 0.6秒  
**关键约束**: 卡片保持原大小 scale(1)，所有效果在卡片上显示

| 阶段 | 时间 | 变化 | 备注 |
|-----|------|------|------|
| 锤痕生成 | 0-20% | 圆形锤痕从0→150px扩大 | 产生凹印感 |
| 碎裂射线 | 20-60% | 8条不规则线向四周扩散 | 不均匀长度增强破碎感 |
| 消退 | 60-100% | 锤痕+碎裂逐步透明 | 柔和淡出

### 5.4 火弩箭 (ID: 3)

**主题**: 火箭连射燃烧  
**关键字**: 5根火箭，白色穿孔，火焰蔓延

#### 视觉效果详解

发射阶段 (0-0.25秒):
- 5根火箭从受击方卡片上方准位落下
- 火箭呈五角形排列，逐个穿过卡片
- 每根火箭为橙红色梯形，伴随耀眼光晕

穿孔阶段 (0.25-0.6秒):
- 5根火箭依次撞击并穿过卡片中心
- 每次穿孔产生一个橙红色圆形火洞
- 火洞呈五角形排列在卡片表面
- 火洞中心明亮(黄光)，边缘深橙色(烧焦感)

燃烧阶段 (0.4-1.2秒):
- 火焰从5个穿孔处不断向四周蔓延
- 火焰呈径向扩散，从穿孔圆形(0%)→卡片边界(100%)
- 火焰颜色：橙→红→暗红渐变，营造燃烧氛围
- 整个卡片逐步被火焰覆盖

伤害结果:
- 若未死亡：火焰消失，卡片恢复，仅留5个穿孔痕迹(淡)
- 若死亡：卡片整体置灰消失

#### 音效 (总1.5秒)
```javascript
playSound('ultimate', 3)

// 发射音效 (5连发，0-0.5s):
// 高通噪声"咻"：2000Hz → 6000Hz，每次0.15s
// 5次间隔0.1s，音量递增（第1-3次较低，第4-5次突增）

// 抵达音效 (5次，0.5-1.5s):
// 沉闷击中音：150Hz → 50Hz，每次0.2s
// 穿孔时配合小幅爆炸音（mid频段突发）
// 穿孔特色：快速连发感+多次撞击密集轰鸣
```

#### 卡片形变动画 (受击方)
**动画名称**: `react-burn`  
**持续时间**: 0.8秒  
**关键约束**: 卡片保持原大小 scale(1)，所有效果在卡片上显示

| 阶段 | 时间 | 变化 | 说明 |
|-----|------|------|------|
| 穿孔出现 | 0-40% | 5个火洞从小→大扩大，亮度提升 | 穿孔显现感 |
| 火焰蔓延 | 40-80% | 火焰从穿孔向外呈圆形扩散 | 覆盖面积逐步增加 |
| 消退平复 | 80-100% | 火焰与穿孔逐步淡出 | 柔和消退

**特点**: 被穿孔冲击感，卡片保持完整但留下伤痕  
**死亡时**: 80%处置灰，穿孔扩大并产生崩碎效果

**施法方 (react-cast-shake)**  
**动画名称**: `castShake-fire`  
**开始时间**: 0秒（与第一箭发射同步）  
**持续时间**: 0.5秒  
**模式**: 限时抖动（非无限循环），共5-6个抖动周期

| 抖动参数 | 值 |
|---------|----|
| 横向位移 | ±8px |
| 纵向位移 | ±4px |
| 旋转幅度 | ±1.5° |
| 单次时长 | 50-80ms |
| 间隔 | 80ms |
| 强度变化 | 后2-3次递减（缓冲感） |

**效果**: 模拟连发火箭的后坐力，强烈但短暂

**⚠️ 重要**: 火弩箭是最长动画，`finalizeRound()` 延迟设为 **1800ms**

### 5.5 雷鸣斩 (ID: 4)

**主题**: 天雷坠落电击着火  
**关键字**: 3颗天雷球，频闪电击，卡通着火

#### 视觉效果详解

下落阶段 (0-0.5秒):
- 3颗发蓝光的雷火球从屏幕上方依次坠落
- 第1颗先落，间隔0.15秒后第2颗坠落，再隔0.15秒第3颗坠落
- 每颗球呈完美圆形，蓝色放射性光晕，内部有淡白色核心

冲击阶段 (0-0.6秒):
- 每颗雷火球落至卡片中心时触发冲击
- 冲击时卡片产生闪光（亮度1.3x）和反色效果(invert 0.2)
- 卡片快速抖动3-4个周期，模拟被电击

电击闪烁 (0.3-0.8秒):
- 受击方卡片做"反色闪烁"动画
- 闪烁10%-40%反色(invert)，视觉上闪现蓝白光芒
- 给予强烈电击感

伤害结果:
- 若未死亡：闪烁消退，卡片恢复
- 若死亡：卡片整体置灰消失

CSS类: `.thunder-ball-fx`, `.react-strobe`, `@keyframes cardShakeElectric`

## 6. 核心系统实现与设计

### 6.1 音效系统（Web Audio API）

#### 音效类型库
```javascript
playSound(type, param)

type 可选值:
  'start'       // 游戏开始鼓声
  'ready'       // 玩家准备音效
  'drum_heavy'  // 重击鼓声
  'drum_light'  // 轻鼓声（答题倒计时）
  'cutin'       // 拔剑高光音
  'ultimate'    // 大招音效 (param: 0-4)
  'miss'        // 答题失败音效
  'revive'      // 复活音效
  'rankup'      // 段位升级音效 (param: rank)
```

#### 核心实现原理
采用**原始波形合成**而非预制音频文件:
```javascript
// 1. 创建振荡器 (Oscillator)
const osc = ctx.createOscillator();
osc.type = 'sine' | 'square' | 'sawtooth' | 'triangle'

// 2. 设置频率曲线（蛋糕的高低）
osc.frequency.setValueAtTime(始频率, 开始时间)
osc.frequency.exponentialRampToValueAtTime(结束频率, 结束时间)

// 3. 设置音量包络（ADSR）
const gain = ctx.createGain();
gain.gain.setValueAtTime(起始音量, 开始时间)
gain.gain.exponentialRampToValueAtTime(目标音量, 目标时间)

// 4. 噪声滤波（可选）
const filter = ctx.createBiquadFilter();
filter.type = 'lowpass' | 'highpass'
filter.frequency.setValueAtTime(频率值, 时间)
```

#### 音效特征说明

| 类型 | 波形 | 频率变化 | 用途 |
|-----|------|---------|------|
| drum_heavy | 三角波 | 150→30Hz | 低沉击鼓 |
| drum_light | 正弦波 | 300→100Hz | 轻鼓声 |
| cutin | 正弦波 | 4000→8000Hz | 高亮拔剑 |
| 一剑开天门 | 锯齿波 | 3000→80Hz | 快速扫过 |
| 雷鸣 | 正弦波 | 快速振幅调制 | 电流感 |

**阶段1: 受击瞬间 (gameState === 'ultimate_impact')**
- 受击方卡片**不置灰**，正常显示原色
- 只展示VFX特效（裂纹、闪光、火焰等）和音效
- 不应有任何灰度或暗化

**阶段2: 受击判定 (gameState === 'result')**

分两种结果：

**2a. 掉血结果** (削减1点HP，但未死亡)
- 触发 `.hp-loss-pulse` 动画（0.6s）
- 持续时间内：

  - 亮度闪烁 (1 → 1.2 → 1.5 → 1.1 → 1)
  - 红色阴影强度变化
- 卡片完整保留，仅有视觉冲击感
- 血条UI更新，掉落一颗心

**2b. 死亡结果** (HP降至0 或以下)
- 触发 `.death-exit-animation` 动画（0.8s）
- 执行顺序：
  1. 0-40%: 逐步加深置灰 (0% → 60% 灰度) + 亮度递减 (1 → 0.7)
  2. 40-60%: 向下平移开始 (translateY 0 → 5vh)
  3. 60-85%: 加速向下滑出 (translateY 5vh → 80vh)
  4. 85-100%: 完全消失 (translateY 80vh → 120vh, scale 1 → 0.5, opacity 1 → 0)
- 最终状态：浅灰色 (#888) + brightness(0.65) + grayscale(1)
- 卡片完全消失后，进入老虎机选人阶段

**判定切换逻辑代码**:
```javascript
// 仅在result阶段且该方是死亡方时才置灰

const isDeathExiting = shouldBeGrayscale && gameState === 'result';
// 掉血时触发脉冲
const isHPLossing = gameState === 'result' && actionEffect?.side === side 
                    && actionEffect?.type === 'hit' && deadSide !== side;
```

**动画类应用**:
```jsx

     style={{
       backgroundColor: shouldBeGrayscale ? '#888' : color,

```

## 9. 复活机制

### 9.1 复活概率

```javascript
checkRevive(side, isDead, hp) {
  if (!isDead) return false
  
  const score = player.score
  const rank = getRank(score)
  
  const reviveProb = 
    rank === 'gold'   ? 0.5 :  // 50%
    rank === 'silver' ? 0.3 :  // 30%
    rank === 'bronze' ? 0.1 :  // 10%
    0                           // 0%
  
  const used = reviveCount[side]
  if (reviveProb > 0 && used < 5 && Math.random() < reviveProb) {
    return true  // 复活成功
  }
  return false
}
```

### 9.2 复活限制
- **每人最多复活5次** (reviveCount[side] < 5)
- **仅高段位有概率**: 无段位=0%, 铜30%失败, 银70%失败, 金50%失败
- **复活后血量**: 1 HP (恢复到最小值)

### 9.3 复活音效
```javascript
playSound('revive')
// 清爽的复活鸣叫音效
```

### 通用音效调用

```javascript

playSound('ready')           // 玩家准备音效
playSound('drum_heavy')      // 重击鼓声
```

#### 各类音效详解

**开局音效** (`playSound('start')`)
- 方波音阶段下滑频率变化
- 配合鼓声噪声提示游戏开始

**准备就绪** (`playSound('ready')`)
- 正弦波上升扫频 (1200Hz→2000Hz)
- 快速切入淡出，提示玩家已准备

**击鼓推进** (`playSound('drum_heavy')` 和 `playSound('drum_light')`)
- **重鼓**: 低频深沉，0.08s
- **轻鼓**: 中频敏捷，0.05s，用于答题倒计时

**失手音效** (`playSound('miss')`)
- 心脏停跳感的低频音
- 从60Hz开始递减，暗示失败

**段位升级** (`playSound('rankup', rankType)`)
- 中国五声音阶 + 余韵钟鸣
- **金段位**: 8音五声阶梯，最后钟鸣回响最强
- **银段位**: 7音五声阶梯，钟鸣中等
- **铜段位**: 6音五声阶梯，较短钟鸣

**复活音效** (`playSound('revive')`)
- 正弦波上扬 (200Hz→800Hz)
- 混合带通噪声，0.6s，传达再起希望

A: 修改 `COLORS` 数组 (约910行)，或修改CSS中的 `#2C2C2C` 等常用色值
